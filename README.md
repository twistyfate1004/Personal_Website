# Personal Website

This is a purely vibe-coded personal website project. The early-stage development was mainly carried out in Claude Code using GLM-4.7. In the later stage, the project was handed over to Codex with GPT-5.5 for some final polishing work.

The core purpose of this project was to test the coding capabilities of these two coding agents and the two models behind them. All prompts used were just simple one-sentence instructions, and the project was never developed as a serious production project. Therefore, this is not an ideal example of a vibe-coding project.

Tip: Except for this sentence, everything else was generated by AI.

- **Next.js 16** - React framework with App Router

## Getting Started

### Prerequisites

- Node.js 18+ installed
- Git installed (optional, for version control)

### Installation

1. Install dependencies:
```bash
npm install
```

2. Run the development server:
```bash
npm run dev
```

3. Open [http://localhost:3000](http://localhost:3000) in your browser

### Admin Content Management

The content manager is available at `/admin`. It lets you edit profile content,
projects, work experience, education, timeline, skills, interests, status, and
life posts from forms in the browser. Content is stored in SQLite instead of
local JSON files, and the existing `data/*.json` files are imported
automatically the first time the app boots with an empty database.

Before using or deploying the admin area, configure:

```bash
ADMIN_PASSWORD=your-admin-password
ADMIN_SECRET=a-long-random-secret
```

`ADMIN_PASSWORD` is required for login. `ADMIN_SECRET` signs the admin session
cookie and should be unique in production.

You can also choose where the SQLite database lives:

```bash
DATABASE_PATH=./storage/site-content.db
```

For long-term public use, deploy this app on infrastructure with persistent
disk storage. SQLite is a good fit for a single-node deployment or VPS. If you
later move to a serverless platform, the data layer should be switched to an
external database.

### Recommended free deployment: Railway

This project is configured to work on Railway Free because Railway supports
persistent volumes for small projects. On Railway:

1. Create a new project from the GitHub repository.
2. Add environment variables:

```bash
ADMIN_PASSWORD=your-admin-password
ADMIN_SECRET=a-long-random-secret
```

3. Create and attach a Volume to the service.
4. Set the Volume mount path to `/data`.
5. Deploy the app.

At runtime, the app automatically uses `RAILWAY_VOLUME_MOUNT_PATH` when
available, so the SQLite database will be stored on the mounted volume.

Important limits:

- Railway Free currently includes a small monthly usage credit and a 0.5 GB
  volume, so this is best for a personal site with light traffic.
- Render Free is not a good fit for this project because its free web services
  use ephemeral local storage, which would not preserve SQLite data between
  restarts and redeploys.

## Daily Maintenance Guide

This site is designed so you rarely need to touch the code. Here's what you'll typically edit:

### 1. Edit Personal Information

Edit `data/config.json`:

```json
{
  "name": "Your Name",
  "tagline": "Your tagline",
  "bio": "Your detailed bio...",
  "socials": [...]
}
```

### 2. Add a New Blog Post

Create a new file in `content/writing/`:

```
content/writing/2024-12-25-your-post-slug.mdx
```

With frontmatter:

```mdx
---
title: "Your Post Title"
date: "2024-12-25"
summary: "A brief summary"
---

Your content here...
```

That's it! The post will automatically appear on `/writing`.

### 3. Add a Project

Edit `data/projects.json` and add a new object:

```json
{
  "id": "unique-id",
  "title": "Project Name",
  "description": "What it does...",
  "tech": ["Tech1", "Tech2"],
  "links": {
    "github": "https://...",
    "demo": "https://..."
  },
  "featured": true
}
```

### 4. Update Your Status

Edit `data/status.json`:

```json
{
  "text": "What you're working on now...",
  "emoji": "🚀"
}
```

This appears on the homepage under "Now".

### 5. Add Images

Put images in the `public/` directory and reference them:

In Markdown:
```markdown
![Alt text](/image-name.jpg)
```

In JSON config:
```json
{
  "avatar": "/your-avatar.jpg"
}
```

### 6. Update Timeline

Edit `data/timeline.json` to add your experience:

```json
{
  "year": "2023 - Present",
  "title": "Your Job Title",
  "description": "What you did...",
  "type": "work"  // or "education" or "other"
}
```

### Changing Colors

Edit `app/globals.css` - modify the CSS variables in `:root` and `.dark`:

```css
:root {
  --accent: #2563eb;  /* Your accent color */
  ...
}
```

### Adding New Pages

1. Create a new directory in `app/`
2. Add a `page.tsx` file

Example:

```bash
mkdir app/links
# Create app/links/page.tsx
```

### Modifying Components

Components are in `components/`. Most are self-explanatory:

- `components/layout/` - Header, Footer, Container
- `components/home/` - Hero, Status, QuickLinks
- `components/projects/` - ProjectCard
- `components/writing/` - PostCard, MDXContent

## Tips

- **Write in Markdown**: Blog posts support full Markdown syntax
- **MDX**: You can use React components in `.mdx` files if needed
- **Git**: Use Git to track changes - it's easy to revert if something breaks
- **Backups**: Keep backups of your `data/` and `content/` folders

## Troubleshooting

### Build Errors

If you get build errors:

1. Check JSON files are valid (use a JSON validator)
2. Make sure MDX frontmatter is properly formatted
3. Run `npm run build` locally to see errors

### Styles Not Working

1. Clear `.next` folder: `rm -rf .next`
2. Restart dev server

### Images Not Showing

1. Make sure images are in `public/`
2. Reference with leading slash: `/image.jpg`
3. Check filename case (Linux is case-sensitive)
