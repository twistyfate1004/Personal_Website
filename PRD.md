# Requirements Document

## 1. Document Purpose

本文档用于描述 `Growing Margins Template` 当前公开版本的目标、范围、功能需求与后续迭代方向，方便继续开发、协作和开源维护。

## 2. Product Overview

这是一个用于展示个人信息、项目经历、教育背景、工作经历、生活记录与文章内容的个人站模板。项目采用静态内容优先的方式，通过本地 JSON 与 MDX 文件管理内容，不依赖数据库或外部 CMS。

## 3. Product Goals

- 提供一个可快速复用的个人主页模板
- 保持内容维护成本低
- 支持简洁清晰的页面结构
- 保留一定的视觉辨识度
- 支持公开发布前的隐私脱敏

## 4. Target Users

- 想快速上线个人主页的开发者
- 需要作品集网站的学生或求职者
- 想维护轻量博客的个人用户
- 希望在本地文件中直接维护内容的用户

## 5. In Scope

- 封面页展示
- 首页归档信息展示
- 项目列表页
- 工作经历页
- 教育背景页
- 时间线展示
- 生活记录页
- 写作列表页与文章详情页
- 基础主题切换
- 中英双语数据结构
- 隐私脱敏后的公开模板版本

## 6. Out Of Scope

- 用户登录
- 后台管理系统
- 在线编辑器
- 数据库持久化
- 评论系统
- 搜索系统
- 完整的国际化路由方案

## 7. Functional Requirements

### 7.1 Cover Page

- 系统应提供独立封面页作为站点入口
- 封面页应展示站点标题、简介文案与入口按钮
- 点击 `Enter Archive` 后应进入归档主页
- 进入归档页时默认应为暗色主题
- 封面背景应使用非私人示意图片资源

### 7.2 Home / Archive Page

- 系统应展示头像、姓名、标语与座右铭
- 系统应展示个人基础信息卡片
- 系统应展示自我介绍与 opening style 文案
- 系统应展示技能与兴趣列表
- 系统应展示时间线内容

### 7.3 Projects Page

- 系统应展示项目列表
- 每个项目应包含标题、描述、技术栈、链接与精选标记
- 项目数据应从 `data/projects*.json` 读取

### 7.4 Work Experience Page

- 系统应展示工作经历列表
- 每条工作经历应包含公司、职位、时间、地点、描述与技能
- 工作经历数据应从 `data/work-experience*.json` 读取

### 7.5 Education Page

- 系统应展示教育背景列表
- 每条教育记录应包含学校、学位、专业、时间、地点与附加信息
- 教育数据应从 `data/education*.json` 读取

### 7.6 Lives Page

- 系统应展示生活记录列表
- 每条生活记录应包含日期、标题、正文与媒体资源
- 媒体应支持图片与视频两种类型
- 当前公开模板中应使用公开可用或占位素材

### 7.7 Writing Module

- 系统应支持文章列表页
- 系统应支持文章详情页
- 文章内容应从 `content/writing/*.mdx` 读取
- 文章详情页应支持目录展示

### 7.8 Theme Support

- 系统应支持暗色与亮色主题
- 当前公开模板默认主题应为暗色
- 用户应可通过页头按钮切换主题

### 7.9 Language Structure

- 系统应保留中英双语内容结构
- 中文与英文内容应分别由独立 JSON 文件维护
- 当前版本可保留单语言默认策略，但代码结构应可扩展

### 7.10 Privacy-Safe Public Template

- 仓库中的默认文案不应包含真实姓名、电话、邮箱、住址、学校或公司信息
- 仓库中的默认视觉资源不应包含真实私人照片
- 仓库中的默认 logo 不应使用私人专属标识

## 8. Content Requirements

- 所有基础内容应优先从 `data/` 目录读取
- 写作内容应优先从 `content/writing/` 目录读取
- 内容更新后无需修改后端逻辑即可生效

## 9. Non-Functional Requirements

### 9.1 Maintainability

- 内容结构应清晰、可读、易修改
- 组件职责应尽量单一
- 文案与页面结构应尽量解耦

### 9.2 Performance

- 常规页面应可通过 Next.js 静态生成或高效渲染
- 图片资源应控制体积
- 页面首屏应保持较快加载速度

### 9.3 Accessibility

- 基础交互元素应具备合理的语义标签
- 链接与按钮应可被键盘访问
- 图像应具备合适的替代策略

### 9.4 Privacy

- 公开仓库发布前应完成内容脱敏
- 不应将真实私人资产直接提交到公开仓库
- 应提醒维护者检查 Git 历史中的敏感资源

## 10. Technical Constraints

- Node.js 版本要求为 `>=20.9.0`
- 前端框架为 `Next.js 16`
- UI 依赖 `React 19`
- 样式方案为 `Tailwind CSS 4`
- 内容方案为 `JSON + MDX`

## 11. Suggested Future Iterations

- 增加真实可切换的国际化能力
- 增加项目详情页与文章标签系统
- 增加 SEO、OG Image 与 sitemap 配置
- 增加 CMS 或后台管理能力
- 增加部署说明与 CI 校验

## 12. Acceptance Criteria

- README 能帮助新用户在本地成功运行项目
- 新维护者能通过 `data/` 与 `content/` 快速找到内容入口
- 项目默认内容不包含明显个人隐私信息
- 项目默认资源不包含私人 logo 与真实照片
- `npm run build` 应成功通过
